# 江湖残卷 - 卡牌版 策划文档

## 一、游戏概述

| 项目 | 内容 |
|------|------|
| 游戏名称 | 江湖残卷 - 卡牌版 |
| 游戏类型 | Roguelike 卡牌战斗 |
| 平台 | 移动端 Web / PC 浏览器 |
| 核心玩法 | 选择职业 → 构建牌组 → 回合制卡牌战斗 → 通关5章 |

## 二、核心战斗系统

### 2.1 回合流程

```
玩家回合开始
  ├── 能量恢复（基础回能 + 额外加成）
  ├── 抽牌（5张 + 额外抽牌奖励）
  ├── 出牌阶段（消耗能量打出手牌）
  └── 结束回合
敌人回合
  ├── 中毒/Debuff 结算
  ├── 敌人攻击（闪避判定 → 护盾/格挡吸收 → 扣血）
  └── 状态回合递减
新回合开始...
```

### 2.2 核心资源

| 资源 | 说明 |
|------|------|
| 能量 | 出牌消耗，每回合恢复（职业不同） |
| 生命值 | 降至0死亡，跨章节保留 |
| 手牌 | 每回合抽5张，上限10张 |
| 牌组 | 抽牌堆 + 手牌 + 弃牌堆 循环 |
| 银两 | 击杀敌人获得，用于事件 |

### 2.3 伤害公式

```
实际伤害 = 基础伤害 × (1 + 攻击增益) + 剑意叠加
防御减伤 = 伤害 × (1 - 防御/(防御+30))
忽视防御 = 防御 × (1 - 忽视比例 - 破甲debuff)
暴击伤害 = 伤害 × (2 + 暴伤加成)
```

### 2.4 闪避系统

- 玩家闪避率 = 基础闪避 + 闪避Buff
- 敌人闪避率 = 各敌人固定值（5%~25%）
- 真实伤害（mode=2）无视敌人闪避
- 强制暴击无视敌人闪避

```

```

## 六、关卡结构

```
章节1 → 战斗(2敌人) → 选卡/事件 →
章节2 → 战斗(2敌人) → 选卡/事件 →
章节3 → 战斗(2敌人) → 选卡/事件 →
章节4 → 战斗(2敌人) → 选卡/事件 →
章节5 → Boss战 → 通关
```

- 每章结束后从3张卡中选1张加入牌组（或跳过保持纯净）
- 40%几率触发随机事件（回血/加钱/强化等）
- 玩家血量跨章节保留

## 九、数据存储

- 使用 `localStorage` 存储永久进度
- 存储内容：总场次、胜场、最佳章节、已解锁卡牌
- 每局临时状态存储在内存变量 `G` 中

## 十、后续扩展方向

1. **更多敌人类型** — 增加特殊技能的敌人（治疗、召唤、护盾）
2. **商店系统** — 章节间用银两购买强化/回血/删牌
3. **遗物系统** — 全局被动加成（类似杀戮尖塔的遗物）
4. **多Boss路线** — 根据章节路径遇到不同Boss
5. **成就系统** — 解锁特殊卡牌的条件任务
6. **平衡性调优** — 持续监控各职业胜率数据
